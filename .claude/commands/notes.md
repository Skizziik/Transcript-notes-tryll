Ты получил команду на транскрипцию аудио и создание детальных заметок.

## Аргументы

`$ARGUMENTS`

## Выбор аудиофайла

**Если аргумент с путём задан** — используй его.

**Если аргумента нет** — найди аудио автоматически в папке `input/`:

```bash
ls -1 input/*.{m4a,mp3,wav,mp4,mov,opus,aac,flac,aiff,ogg,webm} 2>/dev/null
```

- 0 файлов → скажи пользователю: «Положи аудио в `input/` и запусти `/notes` снова» и **остановись**.
- 1 файл → используй его.
- >1 файла → перечисли их и попроси указать какой, например `/notes input/имя.m4a`. **Остановись.**

## Шаги

### 1. Транскрипция

```bash
.venv/bin/python scripts/transcribe.py "<путь_к_аудио>" --out output
```

Если пользователь указал `--lang XX` — передай через `--language XX`, иначе автоопределение.

Транскрипция занимает время (примерно 0.3–1× длительности аудио, первый запуск качает модель ~3 GB). Жди завершения, не ставь timeout слишком маленьким — для часовой записи может занять 15–30 минут.

После — прочитай `output/<basename>.txt` и сообщи: язык, длительность (из `.json`), число символов транскрипта.

### 2. Заметки

Сохрани в `output/<basename>.notes.md`. Принципы:

- **Основной язык — русский.** Английские термины, имена, инструменты — оставляй как есть (`GitHub`, не «ГитХаб»).
- **Структура** — иерархия `##` / `###` по темам. В начале — короткий блок «О чём» (3–5 пунктов).
- **Детализация, а не саммари.** Сохраняй цифры, имена, команды, примеры, цитаты. Определения терминов оформляй явно: **GraphQL** — query language for APIs…
- **Решения и Action Items** — отдельный раздел, по пунктам, с ответственными и сроками если звучали.
- **Открытые вопросы** — отдельный раздел если что-то осталось неясно/отложено.
- **Сильные цитаты** — `> блок`, по возможности с указанием говорящего.
- **Код/команды** — в ` ``` ` блоках.
- **Не выдумывай.** Плохо расслышанное — `[неразборчиво]` или `[?]`.
- **Не добавляй** «Выводы»/«Заключение», если их не было.

Длина — пропорциональна записи (получасовая встреча → 1–2 страницы плотных заметок).

### 3. Конвертация в .docx

```bash
.venv/bin/python scripts/to_docx.py "output/<basename>.notes.md" -o "output/<basename>.notes.docx"
```

### 4. Отчёт пользователю

- путь к `.docx` (главное)
- путь к `.notes.md` (если захочет править перед docx)
- путь к транскрипту `.txt`
- 2–3 пункта о чём запись, чтобы было видно что обработалось правильно

Всё. Только пайплайн, ничего лишнего.
